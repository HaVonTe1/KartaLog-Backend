# AGENTS.md – Domain Module Guidelines

---
## OVERVIEW
- Pure Kotlin data‑model layer.
- No framework annotations, DI, or I/O.
- Guarantees immutability and business invariants.

## CORE ENTITIES
- **Aggregate roots** live in `app/domain/model`.
- Data classes only; use `val` for fields.
- Ids are value objects (`UserId`, `CardId`).
- Equality based on identity (`@JvmField` for Java interop).
- Example pattern:
  ```kotlin
  data class User(
      val id: UserId,
      val name: String,
      val email: Email
  )
  ```
- Keep domain logic inside methods, not extension functions.

## DESIGN CONVENTIONS
- **Packages**: `com.example.domain.<entity>`.
- **Naming**:
  - Entities & value objects: `PascalCase`.
  - Functions: `camelCase`.
  - Constants: `const val` inside `companion object`.
- **Null‑Safety**: Prefer non‑nullable properties. Use `Option<T>` (`kotlinx.collections.immutable.PersistentList`) when absence is a valid state.
- **Result handling**: Return `Result<T>` or sealed `Either<Error, T>` for operations that can fail.
- **Collections**: Prefer immutable collections (`listOf`, `setOf`).
- **Validations**: Encapsulated in value objects; throw `IllegalArgumentException` if invariant broken.
- **Extension functions**: Only for pure transformations, place in `extensions` package.

## PORTS
- **Outbound ports** reside under `app/domain/port/out`.
- Define interfaces only, no implementation details.
- Naming: `<Something>Port` (e.g., `UserRepositoryPort`).
- Method signatures return domain types or `Result`.
- Example:
  ```kotlin
  interface UserRepositoryPort {
      suspend fun findById(id: UserId): Result<User?>
      suspend fun save(user: User): Result<Unit>
  }
  ```
- Keep ports small; one responsibility per interface.

## LOGGING & VALIDATION
- Domain layer must be free of logging frameworks.
- Use `require` / `check` for internal sanity checks.
- For audit trails, expose events via a `DomainEventPort`.

## NOTES
- No Spring, Ktor, or persistence annotations here.
- Keep module compile‑time independent of external libraries.
- Unit tests live in `app/domain/src/test/kotlin` targeting pure functions.
- When adding new entities, update this file to reflect naming and port conventions.

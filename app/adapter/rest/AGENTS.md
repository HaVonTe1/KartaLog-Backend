# AGENTS.md – REST Adapter (app/adapter/rest)
---
## OVERVIEW
- **Purpose:** Expose the domain via HTTP endpoints.
- **Framework:** Spring Boot (Web MVC) – controllers are Spring `@RestController`s.
- **Structure:** Thin layer; delegates to `application` use‑cases.
- **Generated contracts:** Implements OpenAPI‑generated `*Api` interfaces (e.g., `CardApi`, `DeckApi`).
- **Philosophy:** No business logic, only request/response mapping, validation, and logging.

## REST CONTROLLERS
- Each controller corresponds to a generated API interface.
- Annotated with `@RestController` and request‑mapping derived from OpenAPI paths.
- Method signatures match the generated interface (DTOs, `ResponseEntity`).
- Validate input via Bean Validation (`@Valid`).
- Convert exceptions to appropriate HTTP status codes using `@ControllerAdvice`.

## GENERATED OPENAPI INTERFACES
- Code generated by `openapi-generator-maven-plugin` under `target/generated-sources/openapi`.
- Interfaces are pure Kotlin/Java contracts – no Spring annotations.
- Adapter implements them, supplying Spring annotations locally.
- Keep generated code untouched; regeneration safe when API evolves.

## MAPPERS
- **DTO ↔ Domain:** Simple mapper functions (Kotlin extension functions or static Java utils).
- **Naming:** `toDomain()` / `toDto()`.
- **Null‑safety:** Prefer non‑null DTO fields; use `requireNotNull` for mandatory data.
- **Collections:** Map with `map { it.toDomain() }` preserving order.
- **Error handling:** Throw `IllegalArgumentException` for illegal state; let `@ControllerAdvice` translate.

## LOGGING
- Use SLF4J: `private val logger = LoggerFactory.getLogger(this::class.java)`.
- Log at **INFO** for entry/exit of each endpoint, include request identifiers.
- Log **DEBUG** for payloads; guard with `logger.isDebugEnabled`.
- Log **WARN** for recoverable client errors; **ERROR** for unexpected failures.
- Correlate logs via `X-Request-Id` header propagated from controller to service layer.

## NOTES
- Keep controllers **stateless** – inject services via constructor.
- Avoid direct repository access; go through `application` layer.
- Do not add business rules here; defer to domain/use‑cases.
- When OpenAPI changes, regenerate interfaces and adjust mapper signatures accordingly.
- Run `mvn test` – controllers are covered by integration tests in `app/adapter/rest/src/test`.
- lint with `mvn checkstyle:check` and `mvn detekt:check` to keep style consistent.

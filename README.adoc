== Overview
TCGWatcher Backend is a Kotlin‑based service that aggregates trading‑card‑game data from various web sources. It provides a clean Spring Boot REST API, persists data with PostgreSQL (via Testcontainers for integration tests), and includes a robust web‑scraping layer built with Playwright and Jsoup.

== Tech Stack
* **Language**: Kotlin 2.2.21 (JVM 17)
* **Framework**: Spring Boot 4.0.2 (WebMVC)
* **Build**: Gradle Kotlin DSL
* **Persistence**: Spring Data JPA, Hibernate Envers, Liquibase
* **Database**: PostgreSQL (runtime) / SQLite (embedded)
* **Web Scraping**: Playwright 1.58.0, Jsoup 1.22.1
* **Resilience**: Resilience4j 2.3.0
* **Logging**: Kotlin‑logging 7.0.14
* **Testing**: JUnit 5, MockK, Testcontainers, WireMock, Playwright

== Getting Started
=== Prerequisites
* JDK 17
* Docker (required for Testcontainers)
* Gradle (wrapper is included, no separate installation needed)

=== Clone the Repository
[source, bash]
----
git clone https://github.com/<your‑username>/TCGWatcher-Backend.git
cd TCGWatcher-Backend
----

=== Build
[source, bash]
----
./gradlew clean build
----
The build will generate the OpenAPI client, compile the source, and run all unit and integration tests.

=== Run the Application
[source, bash]
----
./gradlew bootRun
----
The service starts on `http://localhost:8080`. Swagger UI is available at `http://localhost:8080/swagger-ui.html`.

=== Running Tests
*All tests*:
[source, bash]
----
./gradlew test
----
*Only unit tests* (exclude integration tests):
[source, bash]
----
./gradlew test --exclude-tags integration
----
*Only integration tests*:
[source, bash]
----
./gradlew integrationTest
----

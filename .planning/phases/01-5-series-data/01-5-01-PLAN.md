# Phase 1.5: Series Data - Plan 01

## Objective

Extend CardMarket scraping to include series information for products.

**Purpose:** The Product domain model already supports series, and ProductDTO already has seriesId/seriesName fields. However, the CardMarket scraper doesn't populate series data when mapping from scraped HTML. This plan adds series scraping and mapping.

**Output:**
- CardmarketProductGallaryItemDto includes series data
- CardmarketProductDetailsDto includes series data
- CardMarketProductMapper maps series to Product domain objects

### Current State

**Product domain model (already exists):**
```kotlin
data class ProductSeries(
    val seriesId: Long,
    val names: Map<String, String> = emptyMap(),

    val series: ProductSeries? = null,
    // ...
)
```

**ProductDTO (already generated):**
```kotlin

    val seriesId: Long? = null,
    val seriesName: String? = null,
    // ...

```

**CollectablesMapper (already maps):**
```kotlin

```

**CardMarketProductMapper (currently MISSING series):**
- toProducts() creates Product objects but sets `series = null`

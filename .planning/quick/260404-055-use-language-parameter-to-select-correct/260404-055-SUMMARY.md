# Quick Task 260404-055: Use Language Parameter to Select Correct TranslationMap Summary

## Objective

Use the language parameter to select the correct TranslationMap in CardMarketDetailsParser for label-based lookups (rarity, release date, price, price trend).

## What Was Done

Added language-aware label selection to CardMarketDetailsParser:

1. **Added `selectLabels` helper function** — Maps lang code (de, en, fr, it, es, pt, nl, pl) to the corresponding Labels from TranslationMap, with English as fallback.

2. **Updated extraction methods** — Modified `extractRarity`, `extractSetInfo`, `extractPrice`, and `extractPriceTrend` to accept a Labels parameter and use `labels.rarityLabel`, `labels.releaseDateLabel`, `labels.priceLabel`, and `labels.priceTrendLabel` instead of hardcoded fallback chains.

3. **Updated parser signature** — Added `translationMap: TranslationMap = DEFAULT_TRANSLATION_MAP` parameter to `parse()` method.

4. **Updated scraper adapter** — CardMarketScraperAdapter now passes `DEFAULT_TRANSLATION_MAP` to `detailsParser.parse()`.
